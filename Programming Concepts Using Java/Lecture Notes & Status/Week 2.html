The general syntax for a format specifier is: %[flags][width][.precision]conversion

  1. Conversion Characters
    • d (decimal integer): For byte, short, int, long, BigInteger.
      • System.out.printf("Integer: %d%n", 123); // Output: Integer: 123
    • s (String): For String objects.
      • System.out.printf("Name: %s%n", "Alice"); // Output: Name: Alice
    • f (floating-point decimal): For float, double, BigDecimal.
      • System.out.printf("Price: %f%n", 19.99); // Output: Price: 19.990000
    • c (character): For char.
      • System.out.printf("Grade: %c%n", 'A'); // Output: Grade: A
    • b (boolean): For boolean. Prints "true" or "false".
      • System.out.printf("Is active: %b%n", true); // Output: Is active: true
    • n (newline): Prints a platform-specific newline character. It's generally preferred over \n in printf as it adapts to different operating systems.
      • System.out.printf("Line 1%nLine 2%n");
        /* Output:
        Line 1
        Line 2
        */
  1. Flags
    • - (Left-Justification): Aligns the output to the left within the specified width.
      • (Without -, it's right-justified by default for numbers, and left-justified for strings if no width is specified, but becomes right-justified if width is specified for strings too).
      System.out.printf("Left-aligned string: %-10s%n", "Hello"); // Output: Left-aligned string: Hello
System.out.printf("Left-aligned number: %-5d%n", 42);     // Output: Left-aligned number: 42
    • 0 (Zero-Padding): Pads numeric output with leading zeros up to the specified width. (Only works with d, f, etc.)
      System.out.printf("Zero-padded: %05d%n", 123); // Output: Zero-padded: 00123
    • , (Thousands Separator): Adds locale-specific thousands separators to numbers.
      System.out.printf("Big number: %,d%n", 1234567); // Output: Big number: 1,234,567 (for US locale)
  1. Width

This is a non-negative decimal integer indicating the minimum number of characters to be written to the output. If the actual output is shorter, it's padded (with spaces by default, or zeros if 0 flag is used).

  • For Strings (s):
System.out.printf("Padded String: %10s%n", "Java"); // Output: Padded String:       Java (right-justified by default)
System.out.printf("Padded String: %-10s%n", "Java"); // Output: Padded String: Java      (left-justified)
  • For Integers (d):
System.out.printf("Padded Integer: %5d%n", 42);  // Output: Padded Integer:    42
System.out.printf("Padded Integer: %05d%n", 42); // Output: Padded Integer: 00042

  1. Precision
    • For floating-point numbers (f): Specifies the number of digits after the decimal point.
    System.out.printf("Formatted Price: %.2f%n", 19.9987); // Output: Formatted Price: 20.00 (rounds)
System.out.printf("Formatted Price: %.1f%n", 19.99);   // Output: Formatted Price: 20.0
    • For strings (s): Specifies the maximum number of characters to be printed. If the string is longer, it will be truncated.
    System.out.printf("Truncated String: %.5s%n", "HelloWorld"); // Output: Truncated String: Hello

import java.util.Scanner;

public class Solution {

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        System.out.println("================================"); // First boundary line

        for(int i = 0; i < 3; i++){
            String s1 = sc.next(); // Reads the string part (e.g., "java")
            int x = sc.nextInt();   // Reads the integer part (e.g., 100)

            // This is the line we're focusing on:
            System.out.printf("%-15s%03d%n", s1, x);
        }
        System.out.println("================================"); // Last boundary line
        sc.close(); // Good practice to close the scanner
    }
}
